--- title: Air Conditioner Protocol layout: protocol ---

Air Conditioner [DRAFT]

Any device that regulates room temperature. Could also be thought of as a temperature actuator.

Overview

Device Type airconditioner
Data Type serialised JSON
Example Modules None
More information

The specification is kept simple so as to cater for the various types of air conditioners that can be integrated into the Ninja Cloud.

Serialised JSON

Parameters
on
boolean
Optional
R+W
 Turn on the airconditioner. `true` is on, `false` is off.
Example: true
tempCurrent
fixed
Optional
R
 The current temperature. Fixed to 1 decimal place.
Only the airconditioner device itself should change this value.
Example: 25.7
tempTarget
fixed
Optional
R+W
 The target temperature for the airconditioner to reach. Fixed to 1 decimal place
Example: 26.3
fanSpeed
number
Optional
R+W
 Set the fan speed setting of the airconditioner starting at 0.
Typically 3 values (0,1,2) however upper limit depends on the airconditioner device
Example: 0
0 LOW
1 MEDIUM
2 HIGH
mode
number
Optional
R+W
 The mode of the airconditioner. The number should match a value in the table below.
Example: 2
0 AUTOMATIC
1 HEAT
2 COOL
3 FAN

Example Unserialised JSON


{
  "on":                   true,
  "tempCurrent":          26.4,
  "tempTarget":           23.1,
  "fanSpeed":             1,
  "mode":                 2,
}
