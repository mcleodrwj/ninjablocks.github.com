Walkthrough

If you're wanting to write your own modules to talk to sensors/actuators, this walkthrough will get you up and running as quickly as possible. See the other Module sub-headings for more comprehensive information.

It is possible to modify code directly on the Ninja Block, or on any machine. If you're directly modifying code within the Ninja Block, start from step 2, and remember to keep backups of your modifications in a directory other than /opt/ninja.

Step 1: Setup Node.js and client code

On any device that can run node follow these instructions. Running node (node client.js) is the equivalent of what the service 'ninjablock' does on the actual Ninja Block hardware.

Note: Disregard errors about /dev/tty01 (arduino). Within the Ninja Block, tty01 is the arduino-cape as seen by the beagle-bone. (If you wish to use an arduino with your client you can see here) For those interested in a high-level overview, see here.

Step 2: Installing ninja-toolbelt and setting up a new module

Follow the install instructions here to install the toolbelt.

Now go to the sub-directory ninja_modules
Note: On the Ninja Block hardware, the client code is in the directory /opt/ninja/, otherwise it is where you checked it out.

When creating your module (ninja create [module name]), follow the instructions for 'ninja create', and then onwards with the instructions for 'npm init'
Note: For 'npm init' most default options should suffice (just hit enter).

If you've not created a README.md file, you will receive a warning, but this can be done later.

Note: if you're running the client code on hardware other than the Ninja Block, restarting the ninjablock service is the equivalent to re-running the client code (node client.js) from the base 'client' directory.

Step 3: Verifying the module template

By default the device ID will be set to 2000 ([client base]/ninja_modules/[module name]/lib/device.js), and assuming that you've already paired you block, this will be presented on the dashboard in a device titled "Sandbox".

What next?

Now you can modify your module to read/write to the hardware that you wish to present as devices.
Then create different device.js files for the devices that you intend to sense/actuate from the Ninja Platform.
